## Known minor limitations (documented, not bugs to chase)

- **Theme `system` mode does not track runtime OS-theme changes.** With theme set to *System*, the app resolves light/dark once (via `matchMedia('(prefers-color-scheme: dark)')`) at load / on each explicit theme change, but there is **no live `matchMedia` change-listener**, so flipping the OS between light and dark while the app is open does not re-theme it until the next reload or manual theme change. Surfaced during the BUG 3 diagnosis (2026-07-14) and deliberately **not** bundled into that fix — it is a separate, latent behavior, not the toggle desync. If it ever becomes annoying it is a one-line fix: add a `matchMedia(...).addEventListener('change', …)` that re-applies the theme while mode is `system`. Recorded here so it is a known behavior rather than a future mystery bug.
